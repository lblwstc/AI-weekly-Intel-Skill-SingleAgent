# Project Instructions

## Skills

A skill is a set of local instructions to follow that is stored in a `SKILL.md` file. Below is the local skill available in this project.

### Available skills

- `build-ai-weekly-intel`: Build end-to-end weekly AI intelligence products from current real-world updates: research the latest releases, synthesize a leadership brief, create a polished internal dashboard, and record a demo video. Use when the user asks for an autonomous AI newsletter, internal intelligence brief, strategy dashboard, demo walkthrough, or a repeatable pipeline that turns recent AI updates into a professional product. (file: `/Users/bolunli/Documents/Practice/11_Newsletter build/ai-weekly-intel/.codex/skills/build-ai-weekly-intel/SKILL.md`)

### How to use skills

- If the user names the skill with `$build-ai-weekly-intel` or asks for the same workflow in plain language, use that skill for the turn.
- Open the skill file first, then load only the specific reference files needed for the request.
- Keep this skill local to the repo. Do not reinstall it into `~/.codex/skills` unless the user explicitly asks for a global install.
